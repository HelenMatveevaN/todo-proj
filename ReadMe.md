# To-Do App (Go + Postgres + Docker)

Простое и эффективное приложение для управления списком задач с использованием слоистой архитектуры (Clean Architecture).

## 🚀 Стек технологий
*   **Backend:** Go 1.21 (библиотеки: `chi` для роутинга, `pgx` для работы с БД)
*   **Database:** PostgreSQL 15 (Alpine version)
*   **Frontend:** Vanilla JavaScript, HTML5, CSS3 (раздается встроенным HTTP-сервером Go)
*   **Infrastructure:** Docker, Docker Compose

## 🏗 Архитектура проекта
Проект разделен на слои для обеспечения масштабируемости и чистоты кода:
*   `cmd/main.go` — инициализация конфига, БД, сервисов и запуск сервера.
*   `internal/handlers/` — HTTP-слой: парсинг JSON, управление роутингом и отправка структурированных ответов.
*   `internal/service/` — бизнес-логика: валидация входных данных (проверка длины заголовка, пустых строк).
*   `internal/database/` — уровень данных: SQL-запросы, миграции и логирование операций.
*   `internal/models/` — описание структур данных (Task, Response).

## ⚙️ Как запустить приложение

### Предварительные требования
Убедитесь, что у вас установлены:
*   **Docker**
*   **Docker Compose**

### Быстрый старт
1. Склонируйте репозиторий.
2. В корне проекта выполните команду:
   ```bash
   docker-compose up --build
3. Приложение будет доступно по адресу: http://localhost:8080

## 📡 API документация
Все ответы API имеют унифицированную структуру:{"data": <payload>, "error": "<message>"}

*   **Метод	Путь	Описание**
*   **GET**	/tasks	Получить список всех задач
*   **POST**	/tasks	Создать новую задачу
*   **PATCH**	/tasks/{id}	Изменить статус выполнения (done)
*   **DELETE**	/tasks/{id}	Удалить задачу из базы
*   **GET**	/health	Проверка работоспособности API

## 🛠 Ключевые возможности
*   **Persistence:** Использование Docker Volumes гарантирует сохранность ваших задач после перезапуска контейнеров.
*   **Validation:** Встроенная проверка в слое Service защищает базу от некорректных данных (пустые заголовки, слишком длинный текст).
*   **Structured Logging:** Все действия с базой данных (создание, удаление, выборка) логируются в консоль Docker с детальной информацией.
*   **Graceful Shutdown:** Сервер перехватывает сигналы прерывания (Ctrl+C) и завершает работу без потери активных соединений.
